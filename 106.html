The following two assignments, for example, are not the same: data[i++] *= 2; data[i++] = data[i++] * 2; 4.12 Evaluation Expressions Like many interpreted languages, JavaScript has the ability to interpret strings of Java‐ Script source code, evaluating them to produce a value. JavaScript does this with the global function eval(): eval("3+2") // => 5 Dynamic evaluation of strings of source code is a powerful language feature that is almost never necessary in practice. If you find yourself using eval(), you should think carefully about whether you really need to use it. In particular, eval() can be a security hole, and you should never pass any string derived from user input to eval(). With a language as complicated as JavaScript, there is no way to sanitize user input to make it safe to use with eval(). Because of these security issues, some web servers use the HTTP “Content-Security-Policy” header to disable eval() for an entire website.
