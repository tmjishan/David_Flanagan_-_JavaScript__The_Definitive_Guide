But as long as the target object is not fro‐ zen, we’ll find that if we can query the proxy with Reflect.isExtensible() and Reflect.getOwnPropertyDescriptor(), and it will tell us that we should be able to set, add, and delete properties. So readOnlyProxy() creates objects in an inconsistent state. We could fix this by adding isExtensible() and getOwnPropertyDescrip tor() handlers, or we can just live with this kind of minor inconsistency. The Proxy handler API allows us to define objects with major inconsistencies, how‐ ever, and in this case, the Proxy class itself will prevent us from creating Proxy objects that are inconsistent in a bad way.
